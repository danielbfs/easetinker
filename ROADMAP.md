## Completed

### Migrate `middleware.ts` to `proxy.ts` (Next.js 16)

Renamed `src/middleware.ts` → `src/proxy.ts`. Body unchanged. The
production build now reports `ƒ Proxy (Middleware)` instead of the
deprecation warning Next.js 16 was emitting.

### Auto-generate technical secrets on first deploy

Implemented in the `init-secrets` compose service. The four technical
secrets (`POSTGRES_PASSWORD`, `NEXTAUTH_SECRET`, `ENCRYPTION_KEY`,
`WORKER_SECRET`) are generated by `docker/init-secrets.sh` on first
boot, persisted in the `secrets` named volume, and loaded into env
vars by `docker/docker-entrypoint.sh`. Postgres uses
`POSTGRES_PASSWORD_FILE` natively. A fresh Hostinger deploy now only
needs `APP_DOMAIN` + OAuth keys filled in.
